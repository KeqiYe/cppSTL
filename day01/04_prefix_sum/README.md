# 第四节：前缀和

前缀和把一段连续区间的求和转换为两个前缀之间的差。

定义 `prefix[i]` 表示前 `i` 个元素之和，则半开区间 `[left, right)` 的和为：

```text
prefix[right] - prefix[left]
```

例如数组：

```text
nums   = [1, 2, 3]
prefix = [0, 1, 3, 6]
```

区间 `nums[1...2]` 的和是：

```text
prefix[3] - prefix[1] = 6 - 1 = 5
```

## LeetCode 560 的核心变形

扫描到当前位置时，设当前前缀和为 `sum`。如果存在一个以前的前缀和
`oldSum`，满足：

```text
sum - oldSum = k
```

那么两个前缀之间的连续子数组之和就是 `k`。移项后得到：

```text
oldSum = sum - k
```

因此可以使用 `unordered_map` 保存“每种前缀和此前出现了多少次”。每次得到新的
`sum` 时：

1. 查询 `sum - k` 出现过多少次，并累加到答案；
2. 再把当前 `sum` 的出现次数加一。

初始化时必须放入：

```cpp
frequency[0] = 1;
```

它表示扫描开始前存在一个和为 0 的空前缀，使得从下标 0 开始、和恰好为 `k` 的
子数组也能被统计。

## 为什么不使用普通滑动窗口

本题数组可能包含负数。右边界加入负数后，窗口和可能减小；左边界移出负数后，
窗口和反而可能增大，因此无法根据窗口和的大小确定应该移动哪个边界。

哈希前缀和解法的预期时间复杂度为 O(n)，额外空间复杂度为 O(n)。为增强整数范围
的安全性，练习中可以用 `long long` 保存前缀和。
